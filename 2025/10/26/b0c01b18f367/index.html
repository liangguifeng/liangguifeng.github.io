如何在没有电信机顶盒的情况下使用电信IPTV服务

犯二青年 发布于 2025-10-26

  • 📒 其他文章
  • 🏷️ IPTV
  • 🏷️ UDP
  • 🏷️ 海南电信

一、前言

近期宽带即将要到期了，于是我就在电信的 APP 上进行续费，在续费时，有看到我当前的套餐是有包含 IPTV 服务的，但是该 IPTV 服务是需要单独购买电信的 IPTV 机顶盒才可以使用的。

由此引发了思考，我一定需要买电信的机顶盒么？IPTV 服务是已经在我的包年套餐里的，理论上 IPTV 机顶盒只能算是增值服务，我不买电信的指定机顶盒，IPTV 的服务我也应该享有才对。

对此我开始查阅大量资料，发现其实电信的 IPTV 是一个单独的内网，光钎入户后，通过光猫进行 VLAN 划分，将该内网划分到了光猫的 IPTV 网口上了，基于此思路，开始了我们今天的折腾。

二、如何操作

1. 获取IPTV内网

首先我们用一条网线将光猫上的 IPTV 网口和电脑连接起来(这里我使用的是 NAS，我有家用的 NAS，并且有双 2.5G 网口，一个接在了主路由上，另外一个网口是空的，目前我把光猫的 IPTV 网口接在了另外一个空闲网口上)，然后打开终端，使用 SSH 连接到 NAS，查看 IPTV 网口 IP 地址，如下图：

我获取到了 IPTV 网口的IP地址为：10.238.143.148，这明显是一个内网 IP 地址，这时候我的 NAS 的网口2应该是进入到了电信的 IPTV 内网。

2. 查找IPTV直播源

进入到 IPTV 的内网后，我们需要找到 IPTV 的直播源，由于我是没有电信机顶盒的，所以我没办法通过抓包的形式获取到 IPTV 的直播源，于是还是开始查阅相关资料。

最终发现，电信 IPTV 的直播源其实并不会经常改变，GitHub 上已经有很多现有的项目开源了各个省份的 IPTV 直播源，我查看的是该项目：https://github.com/wcb1969/iptv，

可以从刚刚我们获取到的 ip 来看，获取到的是一个 V4 的地址，所以我们打开【海南电信组播.txt】文件查看一下，发现里面是 RTP 协议的直播流，我们拿下一个直播源地址，然后尝试播放试试，看看是否正常播放。

这里我是使用 NAS 建了个 Windows 的虚拟机，如果没有这个条件的，可以直接把 IPTV 的网线插到电脑上，然后打开 VLC 尝试播放即可，如图：

播放成功，说明我们获取的 IPTV 的直播源是正确的，接下来就是如何将该直播源代理到我当前的内网中，让我内网的所有设备都可以访问到该直播源。

3. 代理IPTV直播源

由于 IPTV 的直播源是 RTP 协议的，而 RTP 协议的底层是 UDP 协议，这里我们不能直接将 UDP 协议代理到内网中，因为电信的 IPTV 服务是通过组播 (Multicast) 技术传输视频流的

什么是组播：意思是这个视频流不是 发给你个人 的，而是通过 239.x.x.x 这样的组播地址发到整个局域网，凡是加入这个 组 的设备都能收到，也就是所谓的【一对多广播】模式。

如果直接代理，我们局域网内的大多数设备都是不支持组播隔离或优化，也没有配置 IGMP，所以，如果你把组播直接转发或代理出来，那么就会产生下面的情景：

  • 所有局域网设备都会收到这些 IPTV 流量(不管是否在看电视)
  • 每一路 IPTV 都是高码率(比如 8~15 Mbps)
  • 同时多个频道意味着几十甚至上百兆的流量在你内网广播

这时候就产生了【组播风暴】，会导致你的局域网内的网络不断重复、泛洪的组播包，导致整个局域网拥塞、设备卡死、网速极慢。

那么要如何解决呢？

我们使用到了https://github.com/pcherenkov/udpxy工具，来将RTP组播地址变为 HTTP 单播的地址，这样他只订阅你请求的那一路组播，并把它转成普通 HTTP 单播，这样你内网其它设备只看自己需要的流

我的 NAS 是基于 Debian 系统的，所以我使用的工具是https://github.com/valerytschopp/udpxy-debian。

如果你的也是 Debian 系统，可以参考 README 文档，自行编译安装。

如果你的设备是 OpenWrt 系统，那么可以直接在系统软件包中安装。

安装完成后，我们修改默认的配置文件：/etc/default/udpxy

上面的参数解释为：
UDPXY_PORT: 监听的端口，这里监听的是 4022 端口
-a: 监听的IP地址，这里监听的是内网IP地址，也就是 10.0.0.253
-m: 指定接收IPTV组播数据的网卡接口
-l: 日志文件地址
-c: 最大连接数，这里设置为10(因为我光猫的IPTV网卡是百兆的，所以预估单个直播流为10Mbps，那么最大也就支持连接10个客户端同时播放)
-B: UDP 缓冲区大小，单位是字节
-M: 每个客户端的缓存大小，单位是KiB
-v: 启用详细输出，这里我启用了详细输出，这样方便 debug

这时候我们启动代理服务

验证 udpxy 代理是否启动成功，我们可以访问：http://10.0.0.253:4022/status，可以得到如图的状态页面，就是服务启动并代理成功了：

然后我们就可以通过代理服务直接代理 IPTV 的直播源了，以刚刚我们看的下CCTV1的组播地址为例，代理的格式为：http://[udpxy_ip]:[udpxy_port]]/rtp/[组播地址]，我这里的地址为: http://10.0.0.253:4022/rtp/239.253.64.120:5140，打开 IINA 验证效果，如下图：

4. 编写m3u文件

上面我们已经将 IPTV 的组播地址，通过 udpxy 服务代理到我们内网了，但是我们不可能每次都输入一个地址来播放，所以我们可以编写一个 m3u 文件，将所有 IPTV 的直播源都写入这个文件中，这样我们就可以一键播放了。

我们从 GitHub 上的https://github.com/wcb1969/iptv项目已经得到了所有频道的地址，那么这时候我们只需要按照 m3u 文件的格式，对获取到地址进行修改即可。

三、结语

这一次我只是将 IPTV 的直播源代理到局域网内，让每个设备都可以流畅的观看 IPTV，如家里有多台电视，就不需要再单独购买 IPTV 盒子，也不需要电信升级服务了，自己在任意平台就都可以自行播放了。

目前并没有实现的功能如下：

  • IPTV单线融合(也就是上网口和 IPTV 都在一个网口，一条网线即可解决，该功能网上有很多文章，弄起来应该不难，难的是需要光猫的超级密码，我懒得和电信装维师傅扯皮，感兴趣的可以自行探索)
  • 自动生成台标、EPG(目前已知的项目：https://github.com/taksssss/iptv-tool、https://github.com/supzhang/epg)
  • 直播回放功能(如果是udpxy服务代理的单播，可能无法实现回放，可能需要切换为 rtp2httpd 项目才行，但是我认为直播回放功能对我用处不大，需要看指定节目，通过点播平台即可，能够流畅看直播即可)
  • TVBox资源整合(将直播源封装为 TVBox 的资源，方便直接在原版 TVBox 中使用，现在其实已经有一些魔改的软件直接单独设置直播源了，但是官方原版的 TVBox 是不支持单独设置直播源的)
